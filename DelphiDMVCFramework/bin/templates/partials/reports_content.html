Report Generation

Each report runs multiple steps with nested timings. ExeWatch captures the total duration and each sub-step, so you can identify exactly which part is slow. Some reports can randomly fail, producing errors in the dashboard.

Sales Report

4 steps

Heavy report with 4 nested timings: query orders, aggregate revenue, render charts, export PDF. Total: 1-3 seconds.

report.sales report.sales.query_orders report.sales.render_charts

Inventory Report

30% fail rate

Calls a warehouse API that is sometimes down (503). When it fails, the timing is marked as failed and an ERROR is logged.

report.inventory report.inventory.warehouse_api

Audit Report

4 sources

Aggregates data from 4 sources. Slow queries trigger warnings. Compliance checks can find violations (logged as warnings).

report.audit report.audit.transactions report.audit.compliance_check
What to look for in ExeWatch:
  • Timing page — Each report and its sub-steps appear as separate entries. Compare avg/P95 to find bottlenecks.
  • Logs page — Filter by tag "reports" to see INFO, WARNING, and ERROR messages.
  • Breadcrumbs — Click any log entry to see the breadcrumb trail leading to it.
