How ExeWatch Is Integrated

Automatic Request Timing

Every HTTP request is timed via OnBeforeAction / OnAfterAction.

EW.StartTiming(Endpoint, 'http');
// ... request handled ...
EW.EndTiming(Endpoint, nil, Success);

Nested Timings

Reports use nested timings to measure each step inside a parent operation.

EW.StartTiming('report.sales', 'reports');
  EW.StartTiming('report.sales.query', 'db');
  // ...query...
  EW.EndTiming('report.sales.query');
EW.EndTiming('report.sales');

Breadcrumbs

Each request adds a breadcrumb. Service calls add more, creating a trail visible in the log detail.

EW.AddBreadcrumb('GET /web/people', 'http');
EW.AddBreadcrumb('Search: "Ford"', 'people');
EW.AddBreadcrumb('Calling payment API', 'payment');

Counters

Track business metrics: requests, errors, payments, email deliveries, cache hits.

EW.IncrementCounter('http.requests', 1);
EW.IncrementCounter('payment.success', 1);
EW.IncrementCounter('email.rate_limited', 1);

Structured Logging

Every operation is logged with a tag. Use tags to filter on the ExeWatch Logs page.

EW.Info('Payment processed: EUR 149.99', 'payment');
EW.Warning('Slow query: > 1s', 'reports');
EW.ErrorWithException(E, 'email');

Exception Handler

Unhandled exceptions are caught globally and sent to ExeWatch with full details.

fMVC.SetExceptionHandler(
  procedure(E: Exception; ...)
  begin
    EW.ErrorWithException(E, 'unhandled');
  end);
